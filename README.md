# Соцсети-ВСЁ! (SM_BCE)

Небольшой чат на Go и WebSocket с темной темой, общим чатом, личными диалогами, тегами пользователей и сохранением истории сообщений.

Проект сделан как простой мессенджер для локального запуска или размещения на VPS. Сервер написан на Go, интерфейс работает без сборщиков и фреймворков.

## Возможности

- регистрация по имени и уникальному тегу;
- внутренний ID пользователя, поэтому имена могут повторяться;
- общий чат;
- личные диалоги с отдельной историей;
- список пользователей онлайн;
- упоминания через `@tag` в общем чате;
- подсказки тегов при вводе `@` первым символом сообщения;
- уведомления зеленым индикатором для личных сообщений и упоминаний;
- галочка "Запомнить меня";
- сохранение пользователей и сообщений после перезапуска сервера;
- клиентское шифрование сообщений перед отправкой на сервер;
- темная адаптивная тема.

## Требования

- Go 1.22 или новее;
- современный браузер;
- для нормального Web Crypto шифрования: `localhost` или сайт с HTTPS.

Проверить Go:

```powershell
go version
```

## Быстрый запуск

Откройте папку проекта:

```powershell
cd C:\Users\pavel\Documents\Codex\2026-07-01\new-chat\work\SM_BCE
```

Установите зависимости:

```powershell
go mod tidy
```

Запустите сервер:

```powershell
go run ./cmd/server
```

Откройте чат:

```text
http://localhost:8080
```

Если нужно открыть чат с другого устройства в той же сети, используйте локальный IP компьютера:

```text
http://ВАШ_IP:8080
```

Например:

```text
http://192.168.1.15:8080
```

## Локальные данные

При запуске сервер создает рядом с проектом служебные JSON-файлы:

- `users.json` - зарегистрированные пользователи;
- `chat-history.json` - история сообщений;
- `chat-key.json` - текущий ключ чата.

Эти файлы нужны для работы локального экземпляра, но не должны попадать в публичный репозиторий.

## Как работает шифрование

Сообщение шифруется в браузере перед отправкой на сервер. Сервер хранит и пересылает уже зашифрованный текст.

На `localhost` и на HTTPS используется Web Crypto API. Если открыть чат по обычному HTTP через IP в локальной сети, некоторые браузеры могут отключить `crypto.subtle`; для такого случая есть fallback-режим. Он помогает проекту работать в локальной сети, но слабее нормального Web Crypto.

Для реального использования на VPS лучше подключить домен и HTTPS. Тогда браузеры будут использовать полноценный Web Crypto API.

## Запуск на VPS

Базовый порядок:

1. Установить Go на сервер.
2. Скопировать проект.
3. Запустить `go mod tidy`.
4. Запустить `go run ./cmd/server`.
5. Открыть порт `8080` или поставить Nginx/Caddy перед приложением.
6. Подключить HTTPS.

Для постоянной работы лучше запускать приложение через `systemd`, Docker или другой менеджер процессов.

## Важно

Это учебный проект. В нем нет паролей, полноценной авторизации, защиты от спама, ролей администратора и production-базы данных.

Перед реальным публичным запуском стоит добавить:

- вход по паролю или внешнюю авторизацию;
- HTTPS/WSS;
- хранение данных в SQLite или PostgreSQL;
- ограничение размера сообщений;
- защиту от частых запросов;
- резервные копии истории.
