# Соцсети-ВСЁ! (SM_BCE)

Небольшой чат на Go и WebSocket с тёмной темой, общим чатом, личными диалогами, тегами пользователей, E2EE на стороне браузера и сохранением истории сообщений.

Проект можно запускать локально обычной Go-командой, через Docker Compose вместе с PostgreSQL или временно публиковать через Serveo SSH tunnel.

## Требования

- Go 1.22 или новее;
- современный браузер;
- Docker Desktop, если нужен запуск через Docker и PostgreSQL;
- для нормального Web Crypto шифрования: `localhost` или сайт с HTTPS.

## Возможности

- регистрация по имени и уникальному тегу;
- общий чат и отдельные личные диалоги;
- список участников с разделением на онлайн и оффлайн;
- личные сообщения оффлайн-пользователям;
- подсказки тегов через `@` в общем чате;
- зелёные индикаторы новых личных сообщений и упоминаний;
- сохранение пользователей и сообщений после перезапуска;
- шифрование сообщений в браузере перед отправкой на сервер.

## Быстрый запуск без Docker

```powershell
go run ./cmd/server
```

После запуска открой:

```text
http://localhost:8080
```

Если переменная `DATABASE_URL` не задана, сервер использует локальные JSON-файлы:

- `users.json`;
- `chat-history.json`;
- `chat-key.json`.

Эти файлы добавлены в `.gitignore`, потому что это локальные данные.

## Запуск через Docker и PostgreSQL

```powershell
docker compose up --build
```

Приложение будет доступно на:

```text
http://localhost:8080
```

Если порт `8080` уже занят, можно запустить на другом внешнем порту:

```powershell
$env:HOST_PORT=8081
docker compose up --build
```

Тогда сайт откроется на `http://localhost:8081`.

PostgreSQL поднимается отдельным контейнером, а данные сохраняются в Docker volume `postgres-data`.

Чтобы остановить проект:

```powershell
docker compose down
```

Чтобы остановить и удалить базу вместе со всеми сохранёнными сообщениями:

```powershell
docker compose down -v
```

## Публичная ссылка через Serveo

Рабочий обход для быстрого доступа с других устройств без VPS:

```powershell
.\scripts\start-public-chat-serveo.ps1
```

Скрипт запускает локальный Go-сервер на `localhost:8080`, поднимает SSH tunnel через Serveo и выводит публичную HTTPS-ссылку вида:

```text
https://...serveousercontent.com
```

Чтобы остановить локальный сервер и Serveo tunnel:

```powershell
.\scripts\stop-public-chat-serveo.ps1
```

Важно: компьютер должен оставаться включённым, а SSH tunnel должен продолжать работать. Если закрыть туннель или выключить ПК, публичная ссылка перестанет открываться.

## Переменные окружения

`DATABASE_URL` включает хранение в PostgreSQL:

```text
postgres://sm_bce:sm_bce_password@postgres:5432/sm_bce?sslmode=disable
```

`PORT` меняет порт приложения внутри контейнера или локального процесса. По умолчанию используется `8080`.

## Как работает шифрование

Сообщение шифруется в браузере перед отправкой на сервер. Сервер хранит и пересылает уже зашифрованный текст.

На `localhost` и на HTTPS используется Web Crypto API. Если открыть чат по обычному HTTP через IP в локальной сети, некоторые браузеры могут отключить `crypto.subtle`; для такого случая есть fallback-режим. Он помогает проекту работать в локальной сети, но слабее нормального Web Crypto.

Для реального использования лучше подключить домен и HTTPS. Тогда браузеры будут использовать полноценный Web Crypto API.
